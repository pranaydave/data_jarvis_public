DataJarvis.AI · Agent Architecture

ML Pipeline Multi-Agent System

Input
Task
Build breast cancer classifier · accuracy ≥ 0.96
Supervisor
Supervisor Agent
Plans · Evaluates · Routes · Maintains State Memo
Responsibilities
Inspects full pipeline state at each step
Decides next agent and writes targeted instruction
Evaluates model accuracy against threshold (0.96)
Appends reasoning to cumulative supervisor_memo
Triggers feedback loop or advances to testing
End condition: routes to END after testing completes
next_agent instruction supervisor_memo ↑ iteration_count
Sub-Agent 1
Data Exploration
Profiles dataset — activated once at pipeline start
Actions
Loads sklearn breast cancer dataset (569 samples, 30 features)
Computes class distribution, missing values, feature scale ranges
Detects scale imbalance across features
LLM reasons: key risks, preprocessing recommendations, model suggestions
dataset_info class_distribution feature_stats reasoning_log ↑
→ reports dataset_info · supervisor reads and routes to data_prep
Sub-Agent 2
Data Prep
Prepares features — re-runs if accuracy fails threshold
Iteration-Aware Strategy
Pass 1 (iter=0): 10 features only, no scaling — baseline
Pass 2+ (iter≥1): All 30 features + StandardScaler
Stratified 80/20 train/test split each pass
LLM explains choices in context of previous failure
Condition to activate: supervisor routes here when prep needed or accuracy FAIL
prep_info X_train · X_test y_train · y_test scaler (iter≥1)
⚡ direct chain → no supervisor step between prep and training
Sub-Agent 3
Model Training
Trains model and returns accuracy to supervisor
Model Selection (by iteration)
iter=0: Logistic Regression (max_iter=200, unscaled) → ~0.90–0.93
iter≥1: Random Forest (n_estimators=150, scaled) → ~0.97+
Evaluates on held-out test set, produces classification report
LLM analyses: what to improve if below threshold
model_results accuracy (float) passed_threshold classification_report
→ returns model_results to supervisor for evaluation
⚖ Supervisor Evaluates: accuracy vs threshold 0.96
✗ FAIL · accuracy < 0.96
↑
Feedback Loop
iteration_count++
Supervisor writes improved instruction to data_prep:
"Use all features + StandardScaler"

Max iterations: 3 — then forced to testing
✓ PASS · accuracy ≥ 0.96
Sub-Agent 4
Testing
Final evaluation — activated once, triggers END
Outputs
Final accuracy score + full classification report
Confusion matrix (true/false positives and negatives)
Top-5 feature importances (Random Forest)
LLM writes production readiness report
End condition: sets pipeline_complete = True → supervisor routes to END
test_results final_accuracy confusion_matrix top_features
Terminal
END
Pipeline complete · Reasoning log displayed
// Shared State (MLPipelineState)
📊 dataset_info
n_samples, n_features, feature_names, class_distribution, missing_values, scale ranges
🔧 prep_info
description, n_features_used, train_size, test_size, scaling (bool)
🤖 model_results
model_name, accuracy, classification_report, iteration, passed_threshold (bool)
🧪 test_results
final_accuracy, confusion_matrix, top_features, classification_report
💭 supervisor_memo
Cumulative scratchpad — supervisor appends reasoning after every agent call
📋 reasoning_log
Full ordered list of every agent's LLM reasoning — rendered at end of notebook
⚙ control fields
iteration_count · max_iterations (3) · accuracy_threshold (0.96) · pipeline_complete
// Info Exchanged Between Agents
EXPLORATION → SUPERVISOR
Dataset profile: shape, class balance (212 malignant / 357 benign), large feature scale spread — recommendations for preprocessing
SUPERVISOR → DATA PREP
Routing instruction: e.g. "Use all 30 features with StandardScaler — previous model failed at 0.93"
DATA PREP → MODEL TRAINING
X_train, X_test, y_train, y_test (via global data_store) + prep_info dict (description, feature count, scaling flag)
TRAINING → SUPERVISOR
model_name, accuracy (float), classification_report, passed_threshold (bool) — triggers pass/fail decision
TESTING → SUPERVISOR
final_accuracy, confusion_matrix [[TN,FP],[FN,TP]], top-5 feature importances, LLM production report
// Supervisor Routing Logic
  • dataset_info empty → data_exploration
  • prep_info empty → data_prep
  • accuracy ≥ 0.96 → testing
  • accuracy < 0.96 · iter < max → data_prep (iter++)
  • accuracy < 0.96 · iter ≥ max → testing (best effort)
  • test_results exist → end
